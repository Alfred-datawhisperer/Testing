Welcome to Alfred's DevOps Lab

This page is hosted on my Linux VM using NGINX and accessed through a custom local domain.

About This Project

This is my beginner DevOps practice project. I created and hosted this HTML page on a Linux Virtual Machine using NGINX web server. I configured the server to serve this custom page and made it accessible through my local browser.

What I Have Learned So Far

  • How to install and manage NGINX on Ubuntu Linux.
  • How to create and edit an HTML file in the Linux terminal.
  • How NGINX serves files from a defined root directory.
  • How to create a custom server block for a website.
  • How to use a local domain like mysite1.local with the Windows hosts file.
  • How to test NGINX configuration using sudo nginx -t.
  • How to reload NGINX after making configuration changes.
  • How to troubleshoot default page issues caused by wrong server_name settings.

Technologies Used

Linux VM Ubuntu NGINX HTML Hosts File Networking Basics

How This Page Works

When I type http://mysite1.local in my browser, my computer checks the hosts file, finds the IP address of my Linux VM, and sends the request there. NGINX receives the request, matches the domain name in the server block, and serves this index.html file from my website folder.
